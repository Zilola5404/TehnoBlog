# PHP Blog — документация

Кратко:

- Простой блог на «чистом» PHP (без фреймворков) с использованием шаблонизатора Smarty и базой данных MySQL. Проект реализован согласно ТЗ (см. раздел "По какому ТЗ реализовано").

По какому ТЗ реализовано?

Проект реализован по следующему техническому заданию:

Необходимо разработать простой, но полностью рабочий веб-сайт на чистом PHP (без фреймворков) с использованием MySQL и шаблонизатора Smarty. Сайт реализует функционал блога с категориями и постами.

Ключевые требования из ТЗ:

- Категории: название, описание
- Статьи: изображение, название, описание, текст, категории, количество просмотров
- Главная: показать категории с 3 последними постами, кнопка «Все статьи»
- Страница категории: список статей, сортировка, пагинация
- Страница статьи: полная информация + 3 похожих статьи
- Сидинг для наполнения БД
- Без использования фреймворков

Технологический стек

- PHP 8.1+
- Smarty (шаблонизатор)
- MySQL
- SCSS (опционально, для стилей)
- Docker / Docker Compose (окружение)
- Composer (зависимости PHP)
- Node.js / npm (сборка CSS)

Требования (локальная машина)

- Docker и Docker Compose
- Git (опционально)
- PHP 8.1+ (если запуск без Docker)
- MySQL (если запуск без Docker)
- Node.js + npm (для сборки стилей локально)
- Composer (локально, если нужно)

Полная инструкция по запуску (копируйте и выполняйте по порядку)

1. Клонируйте репозиторий:

```bash
git clone https://github.com/Zilola5404/TehnoBlog.git
cd TehnoBlog
```

2. Запустите окружение через Docker (рекомендуется):

```bash
docker compose up -d --build
```

3. Установите npm-зависимости и соберите CSS (локально или внутри контейнера):

Локально:

```bash
npm install
npm run build:css
```

В контейнере (если предпочитаете):

```bash
docker compose exec -T app npm install
docker compose exec -T app npm run build:css
```

4. Заполните базу данных сидером (в контейнере app):

```bash
docker compose exec -T app php database/seed.php
```

5. Очистите кеш шаблонов (локально или в контейнере):

Локально:

```bash
composer clear-cache
```

В контейнере:

```bash
docker compose exec -T app php bin/clear-cache
```

6. Мягко перезапустите Apache (чтобы подхватить изменения конфигурации/шаблонов):

```bash
docker compose exec -T app apachectl -k graceful
```

7. Откройте в браузере:

http://localhost:8080

Если вы не используете Docker — настройте PHP 8.1, MySQL и виртуальный хост вручную и импортируйте `database/schema.sql`.

Архитектура проекта

Структура каталогов (важное):

- `app/` — основной PHP-код (контроллеры, модели, ядро)
  - `Controllers/` — контроллеры (HomeController, PostController, CategoryController)
  - `Core/` — ядро приложения (Router, View, Database)
  - `Models/` — модели (Post, Category)
  - `Support/` — вспомогательные утилиты (Url и т.п.)
- `public/` — публичная директория
  - `assets/` — скомпилированные ресурсы (CSS, JS, img)
  - `index.php` — точка входа
- `templates/` — Smarty-шаблоны (`layout.tpl`, `home.tpl`, `post.tpl`, `category.tpl`)
- `config/` — конфигурация приложения (DB, app)
- `database/` — схема и сиды (`schema.sql`, `seed.php`)
- `assets/scss/` — исходники стилей (SCSS)
- `bin/` — утилиты (очистка кеша)

Пояснения по архитектуре:

- Приложение минималистично, использует паттерн контроллеров и простую модель для работы с БД.
- Smarty отвечает за рендеринг представлений, логика бизнес-правил хранится в `app/`.
- Структура позволяет легко расширять функциональность: добавлять новые контроллеры, модели и шаблоны.

Готовность и тестирование

- Для наполнения тестовыми данными используйте `database/seed.php`.
- Проверьте доступность главной страницы и работоспособность пагинации/сортировки на странице категории.

Советы и частые команды

- Сбилдить CSS при разработке:

```bash
npm run build:css
```

- Перезапустить контейнеры при изменениях Dockerfile / compose:

```bash
docker compose up -d --build
```

- Если что-то пошло не так с правами файлов, проверьте владельца и корректные права для веб-сервера внутри контейнера.
